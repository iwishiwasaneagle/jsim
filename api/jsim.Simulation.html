jsim.Simulation.Simulation module¶

The base object of the interface. The Simulation handles the interaction between the agent and the environment.

A simulation class is created by deriving from Simulation and providing the implementation to collect_data().

class jsim.Simulation.Simulation.Simulation(pa: jsim.Agent.Agent.Agent, pe: jsim.Environment.Environment.Environment)[source]¶

Bases: abc.ABC

Basic class of all simulations. An instance of the simulation is associated with an agent and an environment instances at the moment of creation.

abstract collect_data(ps: jsim.Meta.Sensation.Sensation, pa: jsim.Meta.Action.Action, pnext_s: jsim.Meta.Sensation.Sensation, reward: float) → None[source]¶

This function is called once on each step of the simulation. The default method does nothing. This is where the user can gain access to the simulation’s behaviour.

Parameters

  • ps (Sensation) – Previous sensation, off of which the action was decided

  • pa (Action) – The action taken

  • pnext_s (Sensation) – The sensation from the environment as a result of pa

  • reward (float) – The reward gained by taking action pa

start_trial() → None[source]¶

Forces the beginning of a new trial. Calls self.pa.start_trial() and self.pe.start_trial() to get the first action and sensation respectively.

steps() → None[source]¶

Runs the simulation for num_steps steps, starting from whatever state the environment is in. If the terminal state is reached, it should immediately prepare for the next trail by callied start_trial. Switching from terminal state to a new state does not count as a step.

Parameters

num_steps (int) – Number of steps per trial

trials(max_steps_per_trial: int) → None[source]¶

Runs the simulation for num_trials, starting from whatever state the environment is in. Each trial should be no longer than max_steps_per_trial.

Parameters

  • num_trials (int) – Maximum number of trials

  • max_steps_per_trial (int) – Maximum number of steps per trial
